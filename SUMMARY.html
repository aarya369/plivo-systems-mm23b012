Reliable Streaming Media Transport

This project implements a reliable real-time media transport protocol over UDP. The objective is to minimize playback deadline misses while remaining below the 2× bandwidth overhead limit. The final implementation combines systematic Forward Error Correction (FEC), a playout buffer, duplicate suppression, and deadline-driven scheduling.

Architecture

  • The sender receives media frames from the harness.
  • Frames are divided into FEC blocks containing four source frames.
  • Three parity shards are generated for every block.
  • Both source and parity shards are transmitted through the relay.
  • The receiver reconstructs missing frames whenever any four of the seven shards arrive.
  • Recovered frames are inserted into the playout buffer before playback.

Sender Design

  • Receives 164-byte harness packets.
  • Immediately forwards each source shard.
  • Groups every four frames into one FEC block.
  • Generates three parity shards using GF(28) arithmetic.
  • Serializes packets using network byte order.

Receiver Design

  • Parses incoming wire packets.
  • Stores shards in a fixed-size FEC block ring.
  • Performs matrix inversion over GF(28) when enough shards are available.
  • Places recovered frames into a 512-slot playout buffer.
  • Delivers frames strictly according to playback deadlines.
  • Ignores duplicate packets.

Design Decisions

  • Systematic FEC minimizes latency because original packets can be played immediately.
  • Static buffers provide deterministic memory usage.
  • Single-threaded event loop simplifies synchronization.
  • No retransmissions are used because playback deadlines make round-trip recovery impractical.
  • Network byte-order serialization ensures portability.

Conclusion

The final implementation satisfies the assignment constraints by combining systematic Forward Error Correction, deterministic buffering, and deadline-aware playout scheduling.
